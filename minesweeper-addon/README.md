## Installation

Built for **Jmix 2.8.x** (see `jmixBomVersion` in [`gradle.properties`](gradle.properties) and the `jmix { bomVersion }` block in [`build.gradle`](build.gradle)). Use the **starter** so Boot loads auto-configuration.

| Jmix | Dependency                                                                                                               |
|------|--------------------------------------------------------------------------------------------------------------------------|
| 2.8.x | `io.github.digitilius.minesweeper:minesweeper-starter:<version>` (see `version` in [`gradle.properties`](gradle.properties)) |

```gradle
dependencies {
    implementation 'io.github.digitilius.minesweeper:minesweeper-starter:<version>'
}
```

Use a fixed `<version>` for releases. The add-on registers **`MinesweeperView`** as **`minesweeper_Minesweeper.view`** with route **`minesweeper`**, the **Minesweeper** menu (`io/github/digitilius/minesweeper/menu.xml`), and **Play Minesweeper**. No marker interface on your `MainView` is required: the view uses `@Route(..., layout = DefaultMainViewParent.class)` (`io.jmix.flowui.view.DefaultMainViewParent`) like other Jmix Flow UI views nested under the main shell.

## How to play

- Reveal all **non-mine** cells to **win**. Numbers count adjacent mines (including diagonals).
- **Single-click** a covered cell to reveal (short delay reduces mis-clicks).
- **Double-click** a covered cell to **flag** / unflag. The left display shows **mines − flags**.
- **Smiley** restarts with the same size and mine probability.
- The **timer** starts on the first successful **reveal** (not on a flag-only action).
- After **win** or **loss**, the field stays locked until you change **field size** or hit the smiley.

### Maven Central (Sonatype)

Publishing goes to **Sonatype** (`s01.oss.sonatype.org`); after you close and release the staging repo, artifacts appear on **Maven Central**. Consumers then only need **`mavenCentral()`** and **no token** for public artifacts.

**What you need outside this repo**

1. **Namespace** – Register `io.github.digitilius` (or your chosen `group`) with [Sonatype / Central Portal](https://central.sonatype.com/) and complete verification (DNS or GitHub).
2. **User token** – Create a token in the Sonatype UI; use it as Maven password (username is the token id shown there).
3. **GPG** – Create a signing key, publish the **public** key to a key server, and keep the **private** key for signing (Central requires signatures for release builds).

**Gradle configuration (local: `~/.gradle/gradle.properties` or env; never commit secrets)**

| Property / env | Purpose |
|----------------|---------|
| `sonatypeUsername` / `SONATYPE_USERNAME` | Sonatype user token name |
| `sonatypePassword` / `SONATYPE_PASSWORD` | Sonatype user token secret |
| `signingKey` / `SIGNING_KEY` | Armored **private** GPG key on **one line** in `gradle.properties`: replace every line break with the two characters **`\n`** (backslash + `n`); no real newlines inside the value. `build.gradle` expands those to newlines before signing. |
| `signingPassword` / `SIGNING_PASSWORD` | Passphrase for that key |

When `sonatypeUsername` and `sonatypePassword` are set, `./gradlew publish` uploads to **Sonatype**: snapshot versions go to the **snapshots** repo; release versions use the **staging deploy** URL (`s01.oss.sonatype.org`). When `signingKey` is set, artifacts are **signed** for that publication (expected for Central releases).

**Commands**

```bash
cd minesweeper-addon
./gradlew publish
```

Then use the **Sonatype / Nexus** UI (or Central Portal workflow) to **close** and **release** the staging repository so artifacts sync to Maven Central.

If your account uses only the newer **Central Portal** publishing flow, follow the official [Central Portal + Gradle](https://central.sonatype.org/publish/publish-portal-gradle/) guide; the URLs above are the classic **OSSRH** (`s01`) endpoints, which still match many projects today.

**Consume from Maven Central** (after Sonatype has released the staging repo):

```gradle
repositories {
    mavenCentral()
}

dependencies {
    implementation 'io.github.digitilius.minesweeper:minesweeper-starter:<version>'
}
```

The starter brings in the **`minesweeper`** module and **`MinesweeperAutoConfiguration`**, which imports **`MinesweeperConfiguration`** (Flow UI scanning under `io.github.digitilius.minesweeper`).
